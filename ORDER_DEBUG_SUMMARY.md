## Обнаруженные проблемы

### 1. Отсутствие валидации параметров формы ⚠️

**Проблема:** Форма может отправить пустые или некорректные значения

**Файл:** `/app/apps/dashboard_web/lib/dashboard_web/live/trading_live.ex`

**Что нужно добавить:**
- Валидацию quantity (минимум 0.00001 BTC)
- Валидацию price для LIMIT ордеров
- Проверку формата (только числа)
- Валидацию минимальной суммы ордера (>= 5 USDT)

### 2. Неправильная обработка MARKET ордеров ⚠️

**Проблема:** MARKET ордера включают ненужные параметры (price, timeInForce)

**Решение:** Условно строить параметры в зависимости от типа

### 3. Неинформативные сообщения об ошибках ⚠️

**Проблема:** Ошибки показываются как `inspect(reason)`, что непонятно пользователю

**Решение:** Добавить функцию `format_error_message/1` с расшифровкой

## Типичные ошибки пользователей

### Ошибка: "MIN_NOTIONAL"

**Причина:** Сумма ордера меньше 5 USDT

**Решение:**
```
При BTC = 87745 USDT:

❌ Quantity: 0.00001, Price: 87745
   Сумма: 0.87 USDT (меньше 5)

✅ Quantity: 0.0001, Price: 87745
   Сумма: 8.77 USDT (достаточно)

✅ Quantity: 0.001, Price: 87745
   Сумма: 87.75 USDT (достаточно)
```

### Ошибка: "LOT_SIZE"

**Причина:** Количество меньше минимума

**Решение:**
- BTCUSDT: минимум **0.00001 BTC**
- ETHUSDT: минимум **0.0001 ETH**
- BNBUSDT: минимум **0.01 BNB**

### Ошибка: "Invalid API-key"

**Причина:** Используются production ключи или неправильные ключи

**Решение:**
1. Зайти на https://testnet.binance.vision/
2. Залогиниться через GitHub
3. Создать новые API ключи
4. Обновить `.env`
5. Перезапустить приложение

### Ошибка: "Signature verification failed"

**Причина:** Неправильный secret_key или проблема с timestamp

**Решение:**
```bash
# Проверить secret_key в .env
cat .env | grep BINANCE_SECRET_KEY

# Синхронизировать время (Linux)
sudo ntpdate -s time.nist.gov

# Проверить время
date
```
