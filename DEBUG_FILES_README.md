# README: Файлы для отладки Binance ордеров

Этот документ описывает все файлы, созданные для диагностики и исправления проблем с созданием ордеров на Binance Testnet.

## Краткое резюме

**Статус:** ✅ API работает корректно
**Проблема:** Отсутствие валидации в UI
**Решение:** Готовый патч с исправлениями

## Созданные файлы

### 1. 📋 ORDER_DEBUG_SUMMARY.md
**Назначение:** Краткий итоговый отчет с основными выводами

**Для кого:** Руководители, менеджеры проектов

**Содержание:**
- Результаты тестирования
- Обнаруженные проблемы
- Список всех созданных файлов
- Рекомендации по улучшению
- Проверочный список

**Время чтения:** 5 минут

**Использовать когда:** Нужно быстро понять статус и что делать дальше

**Время чтения:** 30 минут

### 4. 📘 iex_test_commands.md
**Назначение:** Справочник команд для тестирования в IEx

**Для кого:** Разработчики для практического тестирования

**Содержание:**
- Проверка конфигурации
- Получение credentials
- Тесты API вызовов (аккаунт, балансы, цены)
- Создание и отмена ордеров
- Диагностика ошибок
- Альтернативные методы (curl)

**Время чтения:** 15 минут (справочник)

**Использовать когда:** Нужно протестировать API напрямую в IEx

---

### 5. 🔍 test_order_debug.exs
**Назначение:** Автоматический скрипт для полной диагностики

**Для кого:** Все (просто запустить и посмотреть результат)

**Функционал:**
- Проверка конфигурации
- Проверка credentials
- Тест get_account
- Тест get_price
- Тест create_order
- Тест cancel_order
- Анализ ошибок с подсказками

**Использование:**
```bash
mix run test_order_debug.exs
```

**Использовать когда:** Первый шаг диагностики - запустить и увидеть где проблема

---

### 6. 🔧 trading_live_improvements.patch
**Назначение:** Git patch с исправлениями для TradingLive

**Для кого:** Разработчики для применения исправлений

**Содержание:**
- Функции валидации (validate_order_params, validate_quantity, validate_price)
- Правильное построение параметров (build_order_params)
- Информативные сообщения об ошибках (format_error_message)
- Обработка разных типов ордеров (LIMIT, MARKET)

**Применение:**
```bash
git apply trading_live_improvements.patch
```

**Использовать когда:** Готовы применить исправления в код

## Быстрый старт (для нетерпеливых)

```bash
# 1. Проверить что API работает (30 сек)
mix run test_order_debug.exs

# 2. Если все ✅ - применить патч (1 мин)
git apply trading_live_improvements.patch

# 3. Перезапустить сервер (10 сек)
make server

# 4. Тестировать через UI (2 мин)
open http://localhost:4000/trading
```

## Обновление документации

Если что-то изменилось в системе:
